PHASE 6

Global Medication Brain

Phase 6 is the multilingual, jurisdiction-aware, zero-trust Global Medication Brain of Elisence. It safely understands and structures medication behaviour across five languages, powering every major phase of the platform while staying legally compliant with NHS, FDA, EU MDR, and GCC regulations.

1. What Phase 6 Really Is (Honest, Precise)

Phase 6 is the “Global Medication Brain” of Elisence. It is not a medical diagnosis engine and not a drug-recommendation system. It is a strictly controlled, zero-trust semantic engine that understands, structures, and sanity-checks anything users say or type about medications.

This phase converts natural human language (text or voice-transcribed speech) into a safe, structured medical event that downstream systems can trust.

It is already fully built as a stable baseline, and is now evolving into a global drug-intelligence layer that will power:

  • Diabetes
  • Weight-Loss (GLP-1)
  • Women+ Health
  • AHE (Advanced Health Engine)
  • Voice / Zero-Typing (Phase 13)
  • Blind-Assist / Accessibility modules
  • Clinician dashboards (future)

2. What Is Already Built (Verifiable Today)

The current engine in Phase 6 is a real, working baseline. It includes:

2.1 Multilingual Input Normalization

  • Works in EN, FA, AR, TR, RO
  • Cleans text, fixes spacing, unifies digits, normalises casing
  • Prepares tokens for downstream parsing

Implemented & tested — screenshots, code & logs available.

2.2 Spell-Correction & Token Normalization

  • Advanced dictionaries for drug names (e.g., “monjaro”, “mounjaru” → “Mounjaro”)
  • Correction of daily language mistakes across 5 languages

Implemented — in code.

2.3 Drug Detection (Deterministic)

  • Recognises Mounjaro, Ozempic, Wegovy, Metformin and other GLP-1/diabetes medications
  • No hallucination; either a clear detection or "no_match"

Implemented — in code.

2.4 Action Detection (take / stop / miss)

  • Multilingual intent engine with per-language lexicons
  • Negation detection (e.g., “نزنم”, “لا”, “don’t”)
  • Fully deterministic, no LLM involved

Implemented — in code.

2.5 Dose Parsing

  • Regex-based dose extraction (2.5mg, 5 mg, “دو و نیم میلی‌گرم”)
  • Unit normalisation (“mg” unified across languages)

Implemented — in code.

2.6 Date Recognition

  • Recognises ISO dates, UK style, and numeric strings (YYYYMMDD)
  • Rejects ambiguous dates under zero-trust rules

Implemented — in code.

2.7 Semantic Medication Event

Everything above is combined into a structured object:

  • drug_name
  • action
  • dose
  • date_iso
  • warnings
  • confidence
  • uncertainty_reason
  • audit / debug fields

This semantic event backbone is already integrated as a core part of Elisence.

2.8 Phase-Wide API (smart_understand)

  • The unified endpoint for all phases
  • Used by Phase 5 (Elisa), Phase 13 (Voice), Phase 8 (AHE), Diabetes, GLP-1, Women+

Implemented as a stable, single source of truth for medication understanding.

4. Why This Phase Is Legally Safe

Phase 6 is intentionally designed not to cross into diagnosis or treatment.

4.1 NHS

  • Classified as a Health IT component
  • No clinical decision-making
  • Outputs are informational only
  • Every inference is auditable
  • Hazard log is maintained

4.2 FDA

  • Falls under “Non-Device Clinical Decision Support”
  • User (clinician or patient) can independently evaluate the basis of the output
  • No automated treatment recommendations

4.3 EU MDR

  • No standalone medical purpose
  • Does not directly influence treatment decisions
  • A future professional module can be isolated with its own MDR class

4.4 GCC

  • Data residency supported
  • Strong PDPL alignment
  • Non-diagnostic layer → no medical licensing required

4.5 Zero-Trust

  • Every input is treated as untrusted
  • Low-confidence cases trigger clarification
  • Never “guess” a drug or dose
  • No silent assumptions

4.6 BBVP (Build-Back-Verify-Prove)

  • Build – implement the feature
  • Back – create a rollback plan
  • Verify – tests, accuracy checks, fuzzing
  • Prove – evidence pack with SHA-256 hashes

This is what makes Phase 6 regulator-ready.

5. How Phase 6 Supports Voice (Phase 13)

Phase 13 handles speech-to-text and high-level intent classification. Phase 6 interprets the medication content.

Together they enable:

  • “Elisa, I forgot my dose”
  • “الیسا دیشب داروم رو نزدم”
  • “Elisa, increase to 5 mg next week?” → flagged for clinical confirmation

Noise handling:

  • Phase 13 provides an STT confidence score
  • If confidence < threshold → Phase 6 asks for confirmation
  • No uncontrolled interpretations (Zero-Trust preserved)

7. Limitations (Honest & Transparent)

  • Handle all dialects in Arabic or Persian

  • Perform medical triage
  • Provide automated dosing suggestions
  • Replace clinicians

These are intentionally excluded to avoid:

  • High-risk MDR / FDA classification
  • Clinical liability
  • Unsafe or overreaching behaviour
