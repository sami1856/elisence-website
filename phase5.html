1. What Phase 5 Is

It sits above all other phases (Women+, Diabetes, Nutrition, Weight Loss, AHE, Messaging) and provides:

  • Audit-ready evidence for healthcare and government partners.

2. Core Engine – Personality & Message Renderer

The Python engine personality_phase5_engine_v1.py is the Personality & Message Renderer of Phase 5: a deterministic micro-service that converts events and user profile into a localized, styled message object for UI, notifications, and voice.

It receives structured events (e.g. “weight logged”, “safety check completed”, “SOS resolved”) and produces:

PersonalityRenderedMessage { locale, personality_mode, title_text, body_text, cta_text, tone_meta, reward_visual }

Key Components Implemented

  • Supported locales: "en", "fa", "ar", "tr", "ro".
  • Personality modes: calm_gentle, friendly_modern, deep_supportive, adaptive_elisa.
  • Tone sensitivity: soft vs direct, depending on context and risk.
  • Event-based text packs: curated emotional messages for each language.
  • Fallback logic: every request always returns a safe, complete message.
  • Channel adaptation: in-app, notification, or voice.

This engine guarantees Elisa always responds in a predictable, safe, culturally correct way — suitable for clinical, family, and governmental environments.

3. Agentic Coach Architecture & APIs

Perception → Context & Memory → Planning → Actions → Feedback.

From the Phase 5 roadmap, Elisa is implemented as an Agentic Coach built around a closed control loop:

  • Perception: collect signals (weight, steps, mood, meds, risk flags, alerts, schedules).
  • Context & Memory: combine user history, preferences, personality profile, and jurisdiction.
  • Planning: create a plan of action (check-in, advice card, alert, education, nudge).
  • Tool Use / Actions: call APIs in other phases (weight, women’s health, AHE, messaging...).

Core APIs

  • POST /v5/elisa/decide → DecisionRequest → Plan
    Input: user_id, goal, signals (weight, steps_today, mood, meds, last_dose_ts, etc.), history window, constraints (privacy, max API calls, time_budget_ms, risk thresholds).
    Output: structured Plan describing what actions to take, in what order, with which tools.
  • POST /v5/elisa/act → Plan → ActionResult
    Executes operations (send message, schedule check-in, request follow-up, log dopamine events), while recording provenance and safety meta.

Every decision and action is passed through AKAC and safety guards so the coach never bypasses policy, privacy limits, or medical boundaries.

4. Voice Capability – Elisa You Can Talk To

Phase 5 introduces full voice interaction so users can interact with Elisa naturally:

  • Users can speak instead of typing.
  • Elisa responds with natural-sounding speech in the user’s locale.
  • Ideal for walking, running, driving, or visually impaired users.

Voice APIs

  • POST /v5/voice/say → Text-to-Speech (TTS) for Elisa’s messages.
  • POST /v5/voice/transcribe → Speech-to-Text (STT, mock stage).

Voice output uses the same personality rules: gentle, supportive, culturally aligned, and legally safe.

5. Jurisdiction & Legal Context Awareness

Phase 5 never infers critical context just from IP. Instead, Elisa uses Passport V1 (from the cross-phase identity layer) which includes:

  • country_code.
  • jurisdiction_code (e.g. UK_NHS, EU_GDPR, GCC_QATAR).
  • user role (doctor, nurse, patient, parent, coach…).
  • locale and region.
  • organisation (hospital, clinic, ministry).

Elisa’s Safety & Policy Guard uses this to:

  • Switch medical guideline sets (NHS / EU / GCC / WHO).
  • Respect local legal boundaries on advice and phrasing.
  • Handle sensitive topics (women’s health, mental health, pregnancy) correctly.
  • Avoid restricted or unsafe advice.
  • Match cultural tone and expectations.

Localisation is full-stack: language + law + culture + behaviour.

6. Role-Based Communication Styles

Elisa adapts how she speaks depending on who she is talking to:

  • Everyday users: simple, warm, small steps.
  • Doctors: technical, source-linked, guideline-aware.
  • Nurses: care-pattern optimised.
  • Fitness coaches: goal-structured, habit-focused.
  • Pharmacists: medication safety emphasis.
  • Carers/Parents: trauma-aware, supportive, clear.
  • Visually impaired users: high-clarity audio with short, structured cues.

This is controlled by a combination of user profile, personality_mode, event_type, safety guard, locale and culture packs.

8. Blind-Assist Module – “Elisa as Eyes for the User”

One of the most ambitious capabilities linked to Phase 5 is a blind-assist module: a voice-based, real-time “seeing companion” powered by Elisence.

The user can speak naturally: “Elisa, what is in front of me?”

  • Phone camera + Elisence AI analyse objects, obstacles, text, colours, layouts.
  • Elisa returns audio descriptions like: “Chair in front – one metre” or “A table with a cup on the right and an open walkway to the left.”
  • Medication identification, food labels, expiry dates, and documents can be read aloud.
  • Emergency alerts for sudden obstacles or steps.

Technical principles:

  • Real-time, low-latency inference.
  • Local vs cloud processing depending on jurisdiction.
  • Strict privacy (no facial identification, only pattern description).
  • Audio rules tuned for visually impaired users (clear, short, direct).

This module captures the heart of Elisence: technology restoring freedom, dignity, and daily confidence.

10. AI Role vs Rule-Based Logic

Phase 5 is a hybrid system combining deterministic rules with controlled AI:

Rule-Based (Deterministic)

  • Personality packs and tone templates.
  • Locale and language normalisation.
  • Dopamine pacing rules.
  • Safety guard and policy restrictions.
  • Legal and jurisdiction boundaries.
  • Message structure and minimal guarantees.

AI / LLM (Controlled)

  • Deep reasoning when needed.
  • Long-form explanations.
  • Interpreting multiple signals together.
  • Generating natural, human-like responses.
  • Planning multi-step actions across phases.

All AI output flows through the rule-based guard before reaching the user.
